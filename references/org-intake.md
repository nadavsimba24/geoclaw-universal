# Organization Intake Template
Use these prompts during setup so each Geoclaw instance understands the org context and target wins. Capture the answers in `org-profile.yaml` (generated by `scripts/org_intake.sh`) or your CRM.

1. **Organization summary**
   - Name, size, industry, key stakeholders
   - Primary contact + escalation channel
2. **Agent goals**
   - Top 3 objectives (e.g., reduce onboarding time, triage support tickets, summarize reports)
   - Desired KPIs or success metrics
3. **Candidate tasks**
   - Repetitive workflows the org wants automated
   - Source systems involved (email, Slack, Jira, Salesforce, custom DB)
   - Constraints (data residency, approval chains, hours of operation)
4. **User surfaces**
   - Preferred chat apps (Telegram, WhatsApp, Teams, etc.)
   - Required personas/roles (support agent, exec assistant, ops)
5. **Security + compliance**
   - What data the agent can/cannot access
   - Logging/audit requirements
   - API key ownership (shared vs. per-user)

Revisit the intake quarterly to ensure the micro-agent stays aligned with evolving goals.
